# External Integrations

**Analysis Date:** 2025-01-17

## APIs & External Services

**Tencent Cloud - CLB (Cloud Load Balancer):**
- SDK: `github.com/tencentcloud/tencentcloud-sdk-go/tencentcloud/clb` v1.0.1120
- Use: Primary service for port allocation, listener management, and target binding
- Region-aware: Multi-region support via `GetClient(region)` in `pkg/clb/clb.go`
- Rate limiting: Configurable per-API via `pkg/clb/rate-limit.go` (supports limiting on `DescribeLoadBalancers`, `CreateListener`, `DescribeListeners`, `DeleteLoadBalancerListeners`, `BatchRegisterTargets`, `DescribeTargets`, `BatchDeregisterTargets`, `DescribeTaskStatus`)
- Retry strategy: Automatic retry on request limit exceeded (throttling) in `pkg/clb/api.go`
- Batch operations: Optimized batch listener creation (`pkg/clb/batch-listener.go`) and target binding (`pkg/clb/batch-target.go`)
- Client management: Cached per-region in module-level map (`pkg/clb/clb.go`)

**Tencent Cloud - VPC (Virtual Private Cloud):**
- SDK: `github.com/tencentcloud/tencentcloud-sdk-go/tencentcloud/vpc` v1.0.1161
- Use: Optional VPC metadata queries (potential usage for network policy validation)
- Status: Imported but minimal usage in codebase

**Tencent Cloud - CAM (Cloud Access Management):**
- SDK: `github.com/tencentcloud/tencentcloud-sdk-go/tencentcloud/cam` v1.1.3
- Use: Identity and access management (likely for credential verification)

**Tencent Cloud - Tag Service:**
- SDK: `github.com/tencentcloud/tencentcloud-sdk-go/tencentcloud/tag` v1.1.0
- Use: CLB resource tagging for tracking auto-created CLBs
- Tags applied: Cluster ID and port pool name stored for reconciliation

**Metadata Service (for Region Discovery):**
- Used in `pkg/util/region.go` (inferred from `util.GetCurrentRegion()` call)
- TKE node metadata endpoint for automatic region detection if not explicitly provided

## Data Storage

**Kubernetes etcd:**
- Via standard Kubernetes client-go
- Stores: CLBPortPool, CLBPodBinding, CLBNodeBinding CRDs
- Stores: Pod and Node state changes
- Client: `k8s.io/client-go` v0.33.0

**File Storage:**
- Local filesystem only (within container)
- No external S3/OSS integrations

**Caching:**
- In-memory listener cache: `pkg/clb/cache.go` (caches CLB listener info to reduce API calls)
- Port allocation state: In-memory allocator `internal/portpool/allocator.go`
- Client cache: SDK client instances cached per region in `pkg/clb/clb.go`

## Authentication & Identity

**Auth Provider:**
- Custom Tencent Cloud credential management
- Implementation: `pkg/cloudapi/cloudapi.go`
- Credentials: Secret ID + Secret Key (static)
- Initialization: `cloudapi.Init(secretId, secretKey)` called at startup in `cmd/app/manager.go`
- Access: Singleton pattern via `cloudapi.GetCredential()`
- Credential source: Command-line flags or environment variables (`--secret-id`, `--secret-key`)

**Kubernetes RBAC:**
- ServiceAccount: `tke-extend-network-controller` deployed in `kube-system`
- ClusterRole: Auto-generated by kubebuilder, includes permissions for:
  - Reading/writing CLBPortPool, CLBPodBinding, CLBNodeBinding CRDs
  - Watching Pod and Node resources
  - Managing events for audit trail
- RBAC configs: `config/rbac/` generated by `make manifests`

## Monitoring & Observability

**Error Tracking:**
- No external error tracking (Sentry, etc.)
- Error handling: Logger-based error reporting
- Structured logging via `go-logr` + Zap

**Logs:**
- Approach: Structured JSON logging (configurable via Helm `log.encoder: json|console`)
- Log levels: Configurable (debug, info, error, or custom debug levels)
- Destination: stdout/stderr (container logs)
- Key loggers:
  - `setup` - Manager initialization
  - `clb` - CLB API interactions
  - Individual controller loggers

**Metrics:**
- Prometheus metrics endpoint: `:8080/metrics` (or configured via `--metrics-bind-address`)
- Framework: controller-runtime built-in metrics
- Metrics exposed:
  - Controller reconciliation duration
  - Kubernetes API call metrics
  - Standard Go runtime metrics
- ServiceMonitor: Optional Prometheus Operator integration (`config/prometheus/monitor.yaml`)

**Health Checks:**
- Liveness probe: `GET /healthz:8081` (Kubernetes ping check)
- Readiness probe: `GET /readyz:8081` (Kubernetes ping check)
- Probe bind address: Configurable via `--health-probe-bind-address`

**Hosting:**

**CI Pipeline:**
- No external CI/CD system detected

**Deployment Methods:**

## Environment Configuration

**Required Environment Variables (via secrets):**
- `SECRET_ID` - Tencent Cloud API Secret ID
- `SECRET_KEY` - Tencent Cloud API Secret Key
- `REGION` - TKE cluster region (auto-detected if not set)
- `VPCID` - TKE cluster VPC ID (required)
- `CLUSTER_ID` - TKE cluster ID (required)

**Optional Environment Variables:**
- `METRICS_BIND_ADDRESS` - Metrics endpoint address (default: `:8080`)
- `HEALTH_PROBE_BIND_ADDRESS` - Health probe endpoint address (default: `:8081`)
- `LEADER_ELECT` - Enable leader election (default: false → true for HA)
- `WORKER_CLB_POD_BINDING_CONTROLLER` - Pod binding controller concurrency (default: 20)
- `WORKER_CLB_NODE_BINDING_CONTROLLER` - Node binding controller concurrency (default: 20)
- `WORKER_POD_CONTROLLER` - Pod watcher concurrency (default: 20)
- `WORKER_NODE_CONTROLLER` - Node watcher concurrency (default: 20)
- `WORKER_CLB_PORT_POOL_CONTROLLER` - Port pool controller concurrency (default: 10)

**Secrets location:**
- Kubernetes Secret: `tke-extend-network-controller-env` (mounted as `envFrom: secretRef` in Deployment)
- Required fields in secret: `SECRET_ID`, `SECRET_KEY`, `REGION`, `VPCID`, `CLUSTER_ID`

**Rate Limiting Config (via Helm values):**
```yaml
apiRateLimit:
  DescribeLoadBalancers: 20          # requests/sec
  CreateListener: 20
  DescribeListeners: 20
  DeleteLoadBalancerListeners: 20
  BatchRegisterTargets: 20
  DescribeTargets: 20
  BatchDeregisterTargets: 20
  DescribeTaskStatus: 20
```

## Webhooks & Callbacks

**Incoming:**
- Validating Webhook: CLBPortPool resource validation
  - Endpoint: `mutate-clbportpool` (registered in `config/webhook/manifests.yaml`)
  - Type: Mutation webhook (defaulting)
  - Intercepts CLBPortPool creation/update

- Validating Webhook: CLBPortPool resource validation
  - Type: Validation webhook
  - Enforces field immutability rules using `+kubebuilder:validation:XValidation`
  - Path: `internal/webhook/v1alpha1/clbportpool_webhook.go`

**Outgoing:**
- Tencent Cloud API callbacks: None (all CLB operations are synchronous request-response)
- Kubernetes events: Controller records events for audit trail on resource changes

**Event Recording:**
- Event source: `clbportpool-controller` (via controller-runtime EventRecorder)
- Events written to Kubernetes audit log for resource changes (creation, binding, error conditions)
