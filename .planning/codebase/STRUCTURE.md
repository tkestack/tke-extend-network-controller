# Codebase Structure

## Directory Purposes

**api/v1alpha1/**
- Purpose: Define all Kubernetes custom resource types (CRDs) with validation
- Contains: Type definitions for CLBPortPool, CLBPodBinding, CLBNodeBinding, shared CLBBinding spec/status
- Key files:
  - `clbportpool_types.go`: Port pool CRD (cluster-scoped)
  - `clbpodbinding_types.go`: Pod binding CRD (namespaced)
  - `clbnodebinding_types.go`: Node binding CRD (cluster-scoped)
  - `clbbinding_types.go`: Shared binding spec/status types
  - `groupversion_info.go`: CRD group/version metadata
  - `zz_generated.deepcopy.go`: Auto-generated DeepCopy methods (generated)

**cmd/**
- Purpose: Build entry point and manager bootstrapping
- Contains: CLI command setup, flag parsing, controller initialization
- Key files:
  - `main.go`: Binary entry point
  - `app/cmd.go`: Cobra CLI setup, flag definitions
  - `app/manager.go`: runManager() - initialization flow
  - `app/setup_manager.go`: Manager and scheme configuration
  - `app/setup_controller.go`: Controller registration
  - `app/setup_webhook.go`: Webhook registration
  - `app/discovery.go`: Optional scheme discovery (e.g., Agones GameServer)

**config/**
- Purpose: Kubernetes deployment manifests (checked into git)
- Contains:
  - `crd/bases/`: Generated CRD YAML files (generated by kubebuilder)
  - `crd/patches/`: CRD patch overlays
  - `rbac/`: Service account, ClusterRole, ClusterRoleBinding
  - `manager/`: Manager deployment and service configuration
  - `webhook/`: Webhook service and caBundle injection
  - `samples/`: Example CLBPortPool, CLBPodBinding, CLBNodeBinding manifests
  - `certmanager/`: Certificate management for webhooks
  - `default/`: Kustomization overlay combining all configs
  - `prometheus/`: ServiceMonitor for Prometheus scraping

**internal/clbbinding/**
- Purpose: Binding abstraction layer providing unified interface for Pod and Node bindings
- Contains:
  - `clbbinding.go`: CLBBinding interface (polymorphic contract) and Backend interface
  - `clbpodbinding.go`: Pod-specific binding wrapper and podBackend implementation
  - `clbnodebinding.go`: Node-specific binding wrapper and nodeBackend implementation
  - `sort.go`: Comparison functions for binding sorting/deduplication
- Pattern: Wrappers implement CLBBinding interface to enable generic CLBBindingReconciler

**internal/constant/**
- Purpose: Package-level constants used throughout codebase
- Contains: Finalizer names, annotation keys, phase/state constants, magic numbers
- Key: `constant.go` - single file with all constants

**internal/controller/**
- Purpose: Reconciliation logic - implements control loop for each resource type
- Contains:
  - `clbportpool_controller.go`: CLBPortPoolReconciler - manages CLB instances, auto-creation, pre-creation
  - `clbpodbinding_controller.go`: CLBPodBindingReconciler - delegates to generic CLBBindingReconciler
  - `clbnodebinding_controller.go`: CLBNodeBindingReconciler - delegates to generic CLBBindingReconciler
  - `pod_controller.go`: PodReconciler - watches Pods, creates CLBPodBindings
  - `node_controller.go`: NodeReconciler - watches Nodes, manages CLBNodeBindings
  - `gameserverset_controller.go`: GameServerSetReconciler - optional OpenKruiseGame integration
  - `clbbinding.go`: CLBBindingReconciler[T] - generic reconciliation for all binding types
  - `util.go`: Shared reconciliation helpers (finalizers, state transitions)
  - `*_test.go`: Ginkgo test files for each reconciler
  - `suite_test.go`: Shared test setup (envtest)

**internal/portpool/**
- Purpose: Port allocation engine - global state for assigning ports across CLB instances
- Contains:
  - `allocator.go`: PortAllocator singleton - manages multiple PortPools, handles scale-up requests
  - `portpool.go`: PortPool - manages single pool state, implements allocation policies (Uniform/InOrder/Random)
  - `portpools.go`: PortPools type - container for multiple pools
  - `error.go`: Pool-specific error types
  - `portpool_test.go`: PortPool unit tests
- Abstraction: ProtocolPort (port tuple), LBPort (port + LB ID), PortAllocation (lease record)

**internal/webhook/v1alpha1/**
- Purpose: CRD mutation validation via Kubernetes webhooks
- Contains: `clbportpool_webhook.go` - enforces immutability on CLBPortPool spec fields
- Pattern: kubebuilder-generated webhook framework

**pkg/clb/**
- Purpose: Wrapper around Tencent Cloud CLB SDK with caching, batching, retry, rate limiting
- Contains:
  - `clb.go`: Client factory (region-based caching)
  - `api.go`: Direct API call wrappers
  - `listener.go`: Listener lifecycle operations
  - `target.go`: Backend target registration
  - `batch-listener.go`: Batch listener creation
  - `batch-target.go`: Batch target operations
  - `instance.go`: CLB instance creation/deletion
  - `cache.go`: Listener info caching (ListenerCache)
  - `quota.go`: Listener quota tracking (Quota singleton)
  - `param.go`: Parameter validation
  - `wait.go`: Async operation polling
  - `rate-limit.go`: Request throttling (20 qps)
  - `lock.go`: Distributed locking helpers
  - `error.go`: Error classification utilities
- Depends on: `github.com/tencentcloud/tencentcloud-sdk-go/tencentcloud/clb/v20180317`

**pkg/cloudapi/**
- Purpose: Tencent Cloud credentials and authentication setup
- Contains: `cloudapi.go` - credential initialization, GetCredential() factory
- Depends on: Tencent Cloud SDK

**pkg/clusterinfo/**
- Purpose: Cluster-level metadata constants (set at startup)
- Contains: `clusterinfo.go` - ClusterId, VpcId, Region globals, OKGSupported flag
- Usage: Passed via CLI flags at controller startup

**pkg/eventsource/**
- Purpose: Event triggering for controller-runtime reconciliation
- Contains: `eventsource.go` - generic event source for custom event handling
- Usage: Allows non-reconciler code to trigger reconciliation

**pkg/kube/**
- Purpose: Kubernetes API helpers - Pod operations, patching, secret access
- Contains:
  - `pod.go`: Pod finalizer operations, IP lookup
  - `patch.go`: Kubernetes patching helpers
  - `secret.go`: Secret CRUD operations
  - `strip.go`: Strip managed fields from objects
  - `retry.go`: K8s API retry helpers
  - `portpool.go`: PortPool object creation helpers
  - `kube.go`: Client initialization from config
- Pattern: Thin wrappers over controller-runtime client

**pkg/manager/**
- Purpose: Controller-runtime manager configuration
- Contains: `options.go` - NewManager options builder with metrics, probe, leader election setup
- Usage: Called during manager initialization

**pkg/userinfo/**
- Purpose: User account information from Tencent Cloud
- Contains: `userinfo.go` - Init() fetches account ID and other user details
- Usage: Initialize after cloud API setup

**pkg/util/**
- Purpose: General-purpose utilities - pointer ops, collections, retry, env parsing
- Contains:
  - `pointer.go`: Pointer conversion helpers
  - `slice.go`: Slice utility functions
  - `map.go`: Map utility functions
  - `op.go`: Generic operation utilities
  - `retry.go`: Retry logic with exponential backoff
  - `env.go`: Environment variable parsing
  - `region.go`: Region utilities
  - `node.go`: Node-related helpers
  - `*_test.go`: Unit tests
- Pattern: Pure functions with no dependencies

**pkg/vpc/**
- Purpose: VPC SDK client wrapper
- Contains: `client.go` - VPC SDK initialization and wrapper methods
- Usage: Query VPC information during setup

**test/**
- Purpose: Test utilities and E2E tests
- Contains:
  - `utils/utils.go`: Test helper functions
  - `e2e/e2e_test.go`: End-to-end tests
  - `e2e/e2e_suite_test.go`: E2E test suite setup
- Pattern: Ginkgo framework

## Key File Locations

**Entry Points:**
- `cmd/main.go`: Binary entry point - calls app.RootCommand.Execute()
- `cmd/app/cmd.go`: Cobra CLI setup - defines flags and runManager callback

**Configuration:**
- `cmd/app/manager.go`: Manager initialization with cloud credentials and cluster info setup
- `Makefile`: Build targets (build, docker-build, generate, manifests, test, deploy)
- `config/default/kustomization.yaml`: Kustomization overlay for deployment

**Core Logic:**
- `internal/controller/clbbinding.go`: Generic CLBBindingReconciler - the heart of binding reconciliation
- `internal/portpool/allocator.go`: Global PortAllocator singleton - port assignment decisions
- `internal/portpool/portpool.go`: PortPool state machine - per-pool allocation logic
- `pkg/clb/clb.go`: CLB SDK client factory

**Testing:**
- `internal/controller/suite_test.go`: Ginkgo test suite setup with envtest
- `internal/controller/*_test.go`: Individual reconciler tests
- `internal/portpool/portpool_test.go`: Port allocation tests

## Naming Conventions

**Files:**
- `*_types.go`: Kubernetes CRD type definitions (api/)
- `*_controller.go`: Reconciler implementations (internal/controller/)
- `*_webhook.go`: Webhook handlers (internal/webhook/)
- `*_test.go`: Ginkgo test files
- `*_suite_test.go`: Ginkgo suite setup
- `zz_generated.*.go`: Auto-generated by kubebuilder (DeepCopy, manifest generation)

**Directories:**
- `api/`: Public API types
- `internal/`: Private implementation packages
- `pkg/`: Public utility packages
- `cmd/`: Executable entry points
- `config/`: Kubernetes deployment manifests
- `test/`: Test code and fixtures

**Functions:**
- `Reconcile()`: Main reconciliation entry point (reconciler interface requirement)
- `SetupWithManager()`: Reconciler setup/registration
- `sync()`: Reconciliation logic for normal operations
- `cleanup()`: Reconciliation logic for deletion (finalizer cleanup)
- `ensure*()`: State transition helpers (ensureState, ensureBound, ensureUnbound, etc.)
- `Allocate()`: Port allocation operations
- `Release()`: Port deallocation operations

**Types:**
- `*Reconciler`: controller-runtime reconciler type
- `*Pool`: Port pool types
- `CLB*`: CLB-related types
- Interfaces: CLBBinding, Backend, PortAllocatable

## Where to Add New Code

**New Feature (non-binding):**
- Primary code: `internal/controller/new_controller.go` (new reconciler)
- Registration: Add to `cmd/app/setup_controller.go` SetupControllers()
- Types: If CRD needed, add to `api/v1alpha1/new_types.go`
- Tests: `internal/controller/new_controller_test.go`

**New Component/Module:**
- Helper package: `pkg/name/` (if reusable) or `internal/name/` (if private)
- Entry point: Define main exported type/function
- Tests: `pkg/name/name_test.go`

**Utilities:**
- Pod operations: `pkg/kube/pod.go`
- Cloud API wrappers: `pkg/clb/new_feature.go`
- Validation helpers: `pkg/util/new_helpers.go` or `internal/controller/util.go`

**Webhooks:**
- New CRD webhook: `internal/webhook/v1alpha1/newtype_webhook.go`
- Registration: Add to `cmd/app/setup_webhook.go`

## Special Directories

**config/ - Deployment Manifests:**
- Purpose: Kubernetes-native deployment configuration
- Generated: `crd/bases/*.yaml` auto-generated by kubebuilder (do not edit)
- Committed: All files committed to git (expected by deployment scripts)
- Usage: `kubectl apply -k config/default` or `helm install`

**bin/ - Build Outputs:**
- Purpose: Compiled binaries and cached dependencies
- Generated: Yes
- Committed: No (.gitignored)
- Contains: Controller binary, test binaries, k8s test environment

**docs/ - Design and Planning:**
- Purpose: Design documents, API specifications, planning notes
- Contains: API design docs, test plans, architecture diagrams
- Generated: Some files (e.g., `api.md` generated by `make generate-docs`)
- Committed: Yes (except generated docs can be regenerated)

**debug-roc/ - Development Scratch:**
- Purpose: Development and debugging workspaces (NOT production code)
- Generated: User-created test scenarios
- Committed: Yes (for reference)
- Usage: Do not use code from here in production controller
